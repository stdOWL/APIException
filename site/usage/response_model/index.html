    üß© Using ResponseModel¶

    A clean, predictable response structure is the heart of a stable API.

    The ResponseModel in APIException makes sure every success and error response always has the same JSON format ‚Äî easy to document, easy to parse, and friendly for frontend teams.

    ‚úÖ How It Works¶

    Every API response includes:

    • data: your actual payload
    • status: one of SUCCESS, WARNING, FAIL
    • message: a short summary of the outcome
    • error_code: only filled if there‚Äôs an error
    • description: extra context for debugging

    üëâ How to interpret:

    • If data is populated and error_code is null ‚Üí treat as success

    • If error_code is filled ‚Üí treat as fail

    • Your frontend can always rely on the status field to drive logic

    üìå Example¶

    ‚úÖ Import and Use¶

    from fastapi import FastAPI
from APIException import ResponseModel
from pydantic import BaseModel

app = FastAPI()

class UserResponse(BaseModel):
    id: int
    username: str

@app.get("/user")
async def get_user():
    user = UserResponse(id=1, username="John Doe")
    return ResponseModel[UserResponse](
        data=user,
        description="User fetched successfully."
    )

    ‚úÖ Successful Response¶

    "message": "Operation completed successfully.",

    No matter what happens ‚Äî same shape, same fields, always predictable.
